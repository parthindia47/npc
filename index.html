Frequently Asked Questions

  • What is CoreMark?

    CoreMark is a simple yet sophisticated benchmark designed by EEMBC specifically to test the performace of processor core. Running CoreMark produces a single-number score allowing users to make quick comparisons between processors. For more information about CoreMark refer : CoreMark Website

  • How is CoreMark score measured?

    CoreMark score is the number of iterations of CoreMark algorithm that a device executes in 1 second. Unit of CoreMark score is CoreMark.

  • What is CoreMark/MHz?

    CoreMark/MHz is a measure of efficiency of the core.It measured by dividing CoreMark score by the instruction cycle frequency at which the device is running.

  • Where I can see competitors' CoreMark score?

    Competitors' score can be seen at CoreMark Scores Page. Please note that only EEMBC certified scores on above site are the ones which are certified by EEMBC. Rest are all "self-certified" scores.

  • What is Autobench?

    AutoBench is a suite of benchmarks that allow users to predict the performance of microprocessors and microcontrollers in automotive, industrial, and general-purpose applications.It's 16 kernels include basic automotive algorithms (like CAN) and signal processing algorithms(like FFT ,FIR). For more information about Autobench refer : EEMBC Autobench site

  • I can not see individual kernel code size in EEMBC Auto Code size.Which data is shown in graph here?

    EEMBC Autobench can be configured to compile benchmarks with 16 bit data and 32 bit data. Also there are two methodologies for calculating code size depending on whether we want to include test harness size in total code size or not. The data shown in EEMBC Auto Code Size graph is the total size of all benchmarks in Autobench; when each benchmark is compiled separately with Autobench test Harness using 16 bit test data.
